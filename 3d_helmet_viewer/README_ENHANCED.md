# Atlas Enhanced 3D Helmet Frontend

Інтегрований фронтенд для системи Atlas з 3D шоломом, чатом, логами та голосовою взаємодією.

## Особливості

### 🎭 3D Шолом

- Анімований 3D шолом (CSS-based fallback якщо CDN недоступний)
- Відстеження погляду за рухом миші
- Реакція на активність чату та логів
- Природні рухи під час бездіяльності
- Анімація під час розмови (TTS)

### 💬 Чат

- Інтуїтивний інтерфейс спілкування з Атласом
- Розмежування повідомлень користувача та Атласа
- Підтримка голосового вводу (STT)
- Інтеграція з основною системою Atlas

### 📊 Логи

- Відображення логів з усіх контейнерів у реальному часі
- Автоматичне оновлення через WebSocket
- Різні рівні логування (INFO, DEBUG, SUCCESS, WARNING, ERROR)
- Автоочищення старих записів

### 🔊 Голосова взаємодія

- Text-to-Speech (TTS) через MCP сервіс
- Speech-to-Text (STT) для голосового вводу
- Інтеграція з Ukrainian TTS
- Fallback до браузерного TTS

## Файли

- `enhanced_frontend_simple.html` - Основний файл фронтенду
- `enhanced_server.py` - Сервер інтеграції з повним стеком
- `start_enhanced_frontend.sh` - Скрипт запуску
- `Dockerfile` - Контейнеризація
- `requirements-frontend.txt` - Залежності Python

## Запуск

### Простий запуск

```bash
cd 3d_helmet_viewer
./start_enhanced_frontend.sh
```

Або напряму:
```bash
python3 enhanced_server.py
# відкрийте http://localhost:8080/
```

### З повною інтеграцією
```bash
# Запуск з docker-compose (рекомендовано)
docker-compose up atlas-enhanced-frontend

# Або локально з інтеграцією
python3 enhanced_server.py
```

Якщо Docker недоступний/не запущений — сервер автоматично переходить у режим з симульованими логами, TTS/STT мають браузерні fallback-и.

### Тільки статичний сервер
```bash
python3 start_3d_viewer.py
# Потім відкрийте http://localhost:8080/enhanced_frontend_simple.html
```

## Інтеграція з Atlas

### WebSocket API

- `/ws` - Основний WebSocket для real-time комунікації
- Типи повідомлень:
  - `chat` - Повідомлення чату
  - `chat_response` - Відповіді Атласа
  - `logs_update` - Оновлення логів
  - `system_status` - Статус системи

### REST API

- `POST /api/chat` - Надіслати повідомлення до Atlas
- `POST /api/tts/speak` - TTS синтез
- `GET /api/logs` - Отримати логи системи
- `GET /health` - Перевірка здоров'я

### Змінні середовища

- `ATLAS_CORE_URL` - URL основної системи Atlas (default: <http://localhost:8000>)
- `TTS_SERVICE_URL` - URL TTS сервісу (default: <http://localhost:4004>)

## Поведінка шолома

### Відстеження погляду

1. **Миша**: Шолом слідкує за курсором миші
2. **Чат**: При фокусі на чаті - дивиться ліворуч
3. **Логи**: При інтенсивних логах - дивиться праворуч
4. **Бездіяльність**: Природні рухи кожні 5 секунд

### Анімації

- **Розмова**: Легкі рухи під час TTS
- **Активність**: Світіння при взаємодії
- **Ротація**: Постійне обертання в якості базової анімації

## Голосове управління

### Команди

- 🎤 **Voice Input** - Запуск/зупинка голосового вводу
- 🔊 **TTS On/Off** - Увімкнення/вимкнення озвучення

### Підтримувані мови

- Українська (uk-UA) - основна
- Можливість додавання інших мов через TTS MCP

## Технічні деталі

### Браузерна сумісність

- Chrome/Chromium (рекомендовано)
- Firefox
- Safari (обмежена підтримка WebRTC)

### Залежності

- Three.js (для 3D, опційно)
- Model Viewer (для GLB моделей, опційно)
- Web Speech API (для STT/TTS)
- WebSocket API

### Fallback режими

1. **3D модель**: CSS анімація замість WebGL
2. **TTS**: Браузерний TTS замість MCP сервісу
3. **Логи**: Симуляція замість real-time
4. **Чат**: Локальні відповіді замість Atlas API

## Troubleshooting

### Проблема: Шолом не відображається

- Перевірте наявність файлу `DamagedHelmet.glb`
- CSS fallback має працювати завжди

### Проблема: TTS не працює

- Перевірте доступність MCP TTS сервісу на порту 4004
- Браузерний TTS має спрацювати як fallback

### Проблема: Логи не оновлюються

- Перевірте WebSocket з'єднання в DevTools
- API fallback оновлює логи кожні 10 секунд

### Проблема: Голосовий ввід недоступний

- Потрібен HTTPS для production
- Дозвольте доступ до мікрофона в браузері

## Розробка

### Додавання нових функцій

1. Оновіть `enhanced_frontend_simple.html` для UI
2. Додайте API endpoints в `enhanced_server.py`
3. Оновіть WebSocket message handlers

### Кастомізація шолома

- Змініть CSS в секції `.helmet-*` класів
- Додайте нові анімації в `@keyframes`
- Модифікуйте поведінку в методах `*Helmet*`

### Інтеграція з новими сервісами

- Додайте змінні середовища в `enhanced_server.py`
- Створіть нові API методи для взаємодії
- Оновіть WebSocket handlers
