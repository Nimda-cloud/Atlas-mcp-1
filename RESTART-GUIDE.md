# 🔄 Atlas Restart Guide - Post-Conflict Resolution

## Огляд

Цей документ описує вирішення проблеми "конфлікт уже вирішений попробуй перезапустити" (conflict already resolved, try restarting).

## 🆕 Новий скрипт перезапуску

Додано `restart_atlas.sh` - розумний скрипт перезапуску, який автоматично визначає поточний режим роботи Atlas та виконує відповідний перезапуск.

### Основні функції:

- **Автоматичне визначення режиму**: Docker, локальний Python, або Kubernetes
- **Безпечна зупинка**: Коректно завершує всі процеси Atlas
- **Підтримка всіх платформ**: macOS, Linux, Kubernetes
- **Детальна діагностика**: Показує статус сервісів після перезапуску
- **Українська локалізація**: Повні повідомлення українською мовою

## 🚀 Використання

### Рекомендований метод (автоматичний):
```bash
./restart_atlas.sh
```

### Конкретні режими:
```bash
./restart_atlas.sh docker      # Docker Compose
./restart_atlas.sh local       # Локальний Python
./restart_atlas.sh k8s-dev     # Kubernetes development
./restart_atlas.sh k8s-prod    # Kubernetes production
```

### Додаткові опції:
```bash
./restart_atlas.sh --help      # Показати допомогу
./restart_atlas.sh --force     # Примусове завершення процесів
```

## 📋 Сценарії використання

### 1. Після вирішення Git конфлікту
```bash
git status  # Переконайтеся що конфлікт вирішено
./restart_atlas.sh
```

### 2. Перезапуск зависших сервісів
```bash
./restart_atlas.sh --force
```

### 3. Переключення між режимами
```bash
# З Docker на локальний
docker-compose down
./restart_atlas.sh local

# З локального на Docker
./restart_atlas.sh docker
```

### 4. Kubernetes управління
```bash
# Development
./restart_atlas.sh k8s-dev

# Production
./restart_atlas.sh k8s-prod
```

## 🔧 Технічні деталі

### Автоматичне визначення режиму:
1. **Docker**: Перевіряє наявність запущених контейнерів Atlas
2. **Kubernetes**: Шукає namespace `atlas-mcp-dev`
3. **Локальний**: Перевіряє PID файли та процеси Python
4. **Auto**: По замовчуванню використовує Docker

### Безпечна зупинка:
- Завершення по PID файлу (`data/atlas.pid`)
- Вбивання процесів по імені (`atlas_core.py`, etc.)
- Звільнення портів (8000, 8080, 4002-4005)
- Зупинка Docker контейнерів

### Перевірка здоров'я:
- HTTP перевірки endpoints
- Таймаути для кожного сервісу
- Детальне логування результатів

## 🎯 Висновок

Скрипт `restart_atlas.sh` вирішує проблему "конфлікт уже вирішений попробуй перезапустити" шляхом надання:

1. **Єдиного інтерфейсу** для всіх типів перезапуску
2. **Автоматичного визначення** поточного режиму
3. **Безпечного завершення** всіх процесів
4. **Детальної діагностики** стану системи
5. **Українською мовою** для зручності користувача

Тепер користувачі можуть просто виконати `./restart_atlas.sh` після вирішення будь-яких конфліктів і система автоматично перезапуститься в правильному режимі.
